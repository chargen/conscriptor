Locations Where Application Settings Are Stored

QSettings stores settings for an application in up to four locations, depending on whether the settings are user-specific or system-wide and whether the settings are application-specific or organization-wide. For simplicity, we're assuming the organization is called Decade and the application is called Conscriptor.

On Unix systems, if the file format is NativeFormat, the following files are used by default:

  • $HOME/.config/Decade/Conscriptor.conf
  • $HOME/.config/Decade.conf
  • /etc/xdg/Decade/Conscriptor.conf
  • /etc/xdg/Decade.conf

On Mac OS X versions 10.2 and 10.3, these files are used by default:

  • $HOME/Library/Preferences/com.Decade.Conscriptor.plist
  • $HOME/Library/Preferences/com.Decade.plist
  • /Library/Preferences/com.Decade.Conscriptor.plist
  • /Library/Preferences/com.Decade.plist

On Windows, NativeFormat settings are stored in the following registry paths:

  • HKEY_CURRENT_USER\Software\Decade\Conscriptor
  • HKEY_CURRENT_USER\Software\Decade
  • HKEY_LOCAL_MACHINE\Software\Decade\Conscriptor
  • HKEY_LOCAL_MACHINE\Software\Decade

Note: On Windows, for 32-bit programs running in WOW64 mode, settings are stored in the following registry path: HKEY_LOCAL_MACHINE\Software\WOW6432node.

Baud Rate

Select the baud rate for communication with BOB-4's main serial port. Default is 9600 bps. This value must match the rate currently in use by BOB-4, unless communication is through BOB-4's debug port.

You may enter any unlisted value directly into settings.

Baud rate selection is inactive for the debug port because it's fixed by design to 115.2 kbps in BOB-4. In XBOB-4, the debug port is only available internally (not on the rear panel).

Firmware Repair

Debug or Main Port will be selected automatically

You may click the serial icon on the toolbar to confirm connectivity.

This option allows the user to download new firmware even if the current firmware is corrupted and nothing but BOB-4's bootloader is operating. To confirm this condition, connect to either the main or debug serial port at 115.2 kbps using a terminal program. At power up, the BOB-4 bootloader emits: BOB4 boot loader (s: 4.1.2)

In the Serial menu, make sure that the correct PC COM port is selected. The Baud Rate setting doesn't matter in this case because BOB-4's bootloader operates only at 115.2 kbps. Select Serial-->Download-->Firmware (repair). Select the desired BOB-4 firmware file (B4Vxxx.enc) in the Open File dialog when it appears. When the download is complete, BOB-4 resets itself and launches the new firmware.

Terminal

Opens a simple serial communication terminal emulator. using the currently selected PC COM port at the current baud rate setting.
Text typed into the window is line buffered (so corrections are possible) then transmitted when return is pressed, over the serial port, to BOB-4. Messages received from BOB-4 are then displayed.
